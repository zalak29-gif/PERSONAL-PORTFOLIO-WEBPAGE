# GULABI DIARIES - PERSONAL-PORTFOLIO-WEPAGE

*COMPANY*: CODTECH IT SOLUTIONS

*NAME*: PARMAR ZALAK KANTILAL

*DOMAIN*: FRONT END DEVELOPMENT

*DURATION*: 4 WEEKS

*MENTOR*: NEELA SANTOSH

#Gulabi Diaries is currently a personal portfolio-style web project, and based on the folder contents it has two stages:
an initial basic version at the root (index.html) and a significantly refined visual version in gulabi diaries/new/.
The stronger and more complete implementation is clearly the new variant, which presents a full single-page portfolio experience with branded styling, navigation sections, imagery, and responsive layout behavior.

In terms of completed tasks, the project includes a full static site structure with core sections:
Home (hero), About, Projects, and Contact. Navigation links are wired to section anchors, enabling smooth in-page movement.
The hero section introduces identity and positioning (“developer” profile), adds action CTA (“View My Works”), and includes a designed photo composition with layered imagery and decorative elements.
This indicates the project is not just text-based content but intentionally styled for personal branding.

The About section is implemented with descriptive copy and capability tags (analytical thinking, teamwork, adaptability, detail orientation).
The Projects section includes a multi-card grid with project summaries, and the Contact section provides a working mailto link. Footer content is also complete with copyright text.
The code shows deliberate visual direction:
typography pairing from Google Fonts (Cormorant Garamond + Space Grotesk), theme tokens via CSS variables, textured/gradient background treatment, and component-level styling for chips, cards, buttons, and nav pills.

From a frontend engineering perspective, responsiveness is already handled with media queries for tablet and mobile breakpoints.
Layout transitions from two-column hero to stacked flow on smaller screens, nav links wrap for narrow widths, and project cards move from two-column to single-column for readability.
This is a complete responsive static implementation for a portfolio context.

Platform/runtime details: Gulabi Diaries runs as a static website in any browser and does not require a JavaScript runtime, backend API, database, or package manager in its current form.
The root folder also contains an older minimal portfolio file set (index.html, styles.css, empty script.js) that appears to be an earlier draft. The new subfolder is the production-quality version and demonstrates the main completed work.
It can run on XAMPP local hosting or any static host (GitHub Pages, Netlify static deploy, etc.) without build steps.

Languages and technologies are HTML5 and CSS3 (with external font loading).
JavaScript is technically present in the root version (script.js) but currently empty, so the interactive behavior is mostly achieved via HTML/CSS (anchor navigation, hover states, responsive layout, etc.).
Overall, Gulabi Diaries is a completed static personal-brand portfolio with strong visual design and mobile-ready structure.
